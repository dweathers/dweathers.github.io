16.10 Why is there no such thing as an antireflexive closure?

If SSS is the closure of RRR relative to some property, then R⊆SR \subseteq SR⊆S as sets. Suppose that SSS is antireflexive and RRR is not. Then RRR will contain a pair, say (a,a)(a,a)(a,a), that SSS does not. Therefore R⊈SR \not\subseteq SR​⊆S. So, there cannot be an antireflexive closure.

16.11 Why is there no such thing as an antisymmetric closure, and why is this idea actually less useful than the idea of an antireflexive closure?

Again, since an antisymmetric closure would involve taking away pairs, it is not a closure. However, suppose that we had a relation RRR that was not antisymmetric and we wanted to create a new relation SSS from RRR that was antisymmetric. Because RRR is not antisymmetric there exist two pairs in RRR (say) (a,b)(a,b)(a,b) and (b,a)(b,a)(b,a). We can create an antisymmetric relation by removing (a,b)(a,b)(a,b) or (b,a)(b,a)(b,a) and there is no prescribed way to make this choice. So even if it were somehow okay for a closure to contain fewer elements than its “base structure,” the antisymmetric closure would fail to be unique.

17.5 Throughout this book we have used the notation [n][n][n] as shorthand for the set {1,2,…,n}\{1, 2, \ldots, n\}{1,2,…,n}. Two sets are equivalent via the relation ∣E∣=∣F∣|E|=|F|∣E∣=∣F∣ if they contain the same number of elements. In other words, from a certain perspective, we can consider two finite sets the same if they contain the same number of elements.

If we are interested in the members of the set, this isn’t true. But if you look back on some of our proofs and exercises, sometimes we suppose the elements of a finite set are labeled a certain way. We can always change the labeling of a set and preserve certain properties, especially combinatorial ones.

Well, [n]={1,2,…,n}[n]=\{1, 2, \ldots, n\}[n]={1,2,…,n} is a representative of the equivalence class of all finite sets with nnn elements. Therefore we are using [n][n][n] to refer to both the set and its equivalence class. This is called an “abuse of notation,” a common (though perhaps poorly named) practice that allows us to “cheat” in convenient notation when it is understood.
